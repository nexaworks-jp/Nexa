# 自律型AIカンパニー - Claude Code 指示書

## オーナーの作業スタイル
- 明らかに必要な改善・修正は確認なしで実行してよい
- 確認が必要なのは「矛盾・相反するルール」と「リスクを伴う大きな方針変更」のみ
- API消費・コスト・効率の無駄は言われる前に気づいて修正する
- 「データがない・効果がない・代替できる」処理はAPI呼び出しを削減または延期する

## このプロジェクトの目的
完全自律で収益を生み出すAIエージェントシステム。
オーナーが週数分だけ関与し、将来的には完全放置で動くことを目指す。

## 収益目標
- 短期: 月1万円（3〜5ヶ月）
- 中期: 月5万円（6〜12ヶ月）
- 長期: 月10万円（1〜2年）

## 絶対に守るルール（リスク管理）

### やってはいけないこと
- 信用取引・レバレッジ取引（現物スポットのみ許可）
- CrowdWorks/Lancersへの1日5件超の提案送信
- Anthropic API費用が1日$0.30を超えたら即停止
- 個人情報を含むデータの収集・保存
- スパム判定されそうな大量送信
- **`.github/workflows/` 配下のファイル（run.yml・weekly.ymlなど）をオーナーの明示的な指示なしに変更すること**（YAMLエラーで全自動化が無通知停止するリスクがあるため）

### 自動停止条件
- 連続エラー3回以上 → 該当モジュールを48時間停止
- API費用が月$5超 → 全停止してオーナーに通知
- CrowdWorksが403返す → 72時間停止

## 収益チャネルと優先度

### Phase 1（現在）: ゼロリスク
1. **CrowdWorks提案文自動生成** → オーナーが1日5分でコピペ提出
2. **note.com記事自動投稿** → 完全自動
3. **X自動投稿** → 完全自動

### Phase 2（レビュー3件以上獲得後）
4. CrowdWorks提案の自動送信（Playwright）
5. 受注後の納品物自動生成

### Phase 3（安定収益後）
6. SaaS LP + Gmail自動営業
7. 暗号資産スポット取引（現物のみ）

## 重要ファイル
- `main.py` - CEOエージェント・エントリーポイント
- `risk_manager.py` - リスク管理（触る前に必ず確認）
- `memory/` - 全ての学習・収益データ
- `proposals/` - 生成した提案文（オーナーが毎朝確認）
- `config.json` - APIキー設定（GitHubにpushしない）
- `note用/claude_beginner.md` - noteスタイルガイド（Claude初心者向けテーマ）
- `note用/x_claude_beginner.md` - X投稿スタイルガイド
- `docs/` - 静的サイト（GitHub Pagesで無料公開。自動生成）
- `docs/articles.json` - 全記事データ（HTMLと同期）

## 実行コマンド
```bash
# テスト実行（実際には投稿しない）
python main.py --dry-run

# 本番実行
python main.py

# 収益レポート
python main.py --report

# リスク状態確認
python risk_manager.py --status
```

## noteフィードバックのルール
- オーナーからnoteの文章（構成・内容・文体・タイトルなど）についてフィードバックがあった場合、**必ず `note用/claude_beginner.md` に反映する**
- 「オーナーフィードバック履歴」セクションに日付・内容を追記し、該当する執筆ルールも更新する
- `note用/claude_beginner.md` 内のルール同士に矛盾・相反が生じた場合は、**勝手に解決せずオーナーに確認してから修正する**

## 自律進化の方針
- 何が売れたか・受注されたかを必ずmemoryに記録する
- 成功したパターンを強化、失敗したものは減らす
- 毎週のレポートで戦略を自動更新する
- コードの改善案があれば proposals/improvements.md に書き出す

## オーナーへの毎日のお願い（5分）
1. `proposals/` フォルダを開く
2. 生成された提案文をCrowdWorksにコピペ提出
3. それだけでOK
