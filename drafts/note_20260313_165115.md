React コンポーネント\n- `server/` - バックエンド（Express）\n- `tests/` - テストコード\n\n## 開発ルール\n- ESLint: airbnb設定を使用\n- 関数は TypeScript で型定義必須\n- コンポーネントは関数型を使用\n```\n\n#### ARCHITECTURE.md の例\n\n```markdown\n# アーキテクチャ\n\n## データフロー\n1. React Component → Custom Hook（useAuth）\n2. Hook → API（src/utils/api.js）\n3. API → Express Server（server/routes/api.js）\n4. Server → Database\n\n## 重要な設計判断\n- 状態管理: Context API（Redux は使わない）\n- API通信: fetch + custom hook でラップ\n- 認証: JWT（LocalStorage に保存）\n\n## 注意点\n- `node_modules/` 配下は機械生成なので参照しない\n- DB接続情報は .env に記載\n```\n\n---\n\n## 実践的なセットアップ③：Claude Codeへの「初期プロンプト」の工夫\n\n### よくあるNGパターン\n\n```\n「このボタンを赤くしてください」\n```\n\n→ Claude Codeは「どのボタン？」「どのファイル？」って感じで、何度も聞き直します。\n\n### 効果的なプロンプトの書き方\n\n```\n「src/components/Button.jsx の PrimaryButton コンポーネントを、\nクリック時に背景色が #FF5733 に変わるようにしてください。\n\nスタイルは src/styles/colors.js で定義されている COLORS.primary を使い、\nCSS Modules で実装してください。\n\nこの変更後、tests/unit/Button.test.js のテストが全て pass することを確認してください。」\n```\n\n**ポイント:**\n- ファイルパスを明示\n- 何を使うか（ライブラリ・スタイル方式）を指定\n- テストまで含める\n\n---\n\n## 実践的なセットアップ④：コンテキスト効率を上げるコツ\n\n### 1. 複数ファイルの関係性を先に説明する\n\nClaude Codeに「これらのファイルは繋がってる」って伝えてあげます。\n\n```\n「AuthContext.js で管理している isLoggedIn を、\nHeader.jsx で読んで表示する仕様です。\n\nこの二つのファイルを確認してから、\nLogout ボタンのクリックハンドラーを追加してください。」\n```\n\n### 2. 「参照してほしいファイル」を明確にする\n\n```\n「src/hooks/useAuth.js を参照しながら、\nsrc/components/LoginForm.jsx を修正してください。\n\nこのhookの戻り値（login, logout, isLoading）を活用してください。」\n```\n\n### 3. 修正前後で「制約条件」を書く\n\n```\n「以下の制約を守りながら API 通信を実装してください：\n- timeout は 10秒\n- error時は console.error で記録\n- 成功時は React Query で cache される想定\n- TypeScript で型定義必須」\n```\n\n---\n\n## をよく使う\n- 型定義は JSDoc で書く\n\nこれを踏まえて実装してください。」\n```\n\n### ❌ つまずき③：テストコードがない\n\n**症状:**動くコードは出るけど、テストは書かない\n\n**原因:**「テストを書いてほしい」ってプロンプトが曖昧\n\n**解決策：**\n```\n「実装後、tests/unit/ 応用：複数プロジェクトでの再利用テンプレート\n\nセットアップを毎回やるのは面倒なので、テンプレート化するのがオススメです。\n\n```\nproject-template/\n├── README.md\n├── ARCHITECTURE.md\n├── .claudeignore\n├── CLAUDE_SETUP.md          # ← 新規ファイル\n```\n\n####
