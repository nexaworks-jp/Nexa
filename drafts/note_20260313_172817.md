# AIの記憶ってどうなってるの？｜LLMの長期記憶を初心者向けに解説

## AIの記憶ってどうなってるの？｜LLMの長期記憶を初心者向けに解説

「ChatGPTに前に話したことを覚えてるのかな…」「AIはどうやって情報を記憶しているんだろう」と疑問に思ったことはありませんか？

わたしも最初、AIとの会話を重ねていくうちに「あれ、このAI、さっき言ったことを忘れた？」という経験をしました。実は、AIが情報を記憶する仕組みは、サービスによって異なるんです。一部のサービスは会話履歴を自動保持していますが、完全な「個別ユーザーの長期記憶機能」を標準で提供していないものもあります。その理由と、それをどう補うのかについて、わたしが学んだことをシェアします。

## AIの記憶って何のこと？

まず、記憶という言葉を整理しておきましょう。わたしたちが「記憶」というと、昨日何を食べたかとか、10年前の思い出とか、いろんなタイムスパンの情報を思い浮かべますよね。

AIの記憶機構も似たようなもので、大きく分けると2つの概念があります。

**短期記憶（またはコンテキスト保持）**とは、今この瞬間の会話の中での情報のことです。たとえば、あなたがChatGPTに「わたしの名前はタロウです」と言ったら、その直後の返答では「タロウさんへのお返事は…」と名前を使ってくれます。ただし、多くのAIサービスでは、会話のコンテキストウィンドウ（保持できる文字数や会話数に制限がある）があるため、非常に長い会話になると古い情報を失うことがあります。

**長期記憶**とは、複数の会話や長い期間にわたって情報を保持することです。わたしたちが「あ、この人は毎月の家計簿が悩みなんだったな。だから今月の支出管理についてアドバイスしよう」と思うような、背景知識を持った対応のことです。

## なぜAIは完全な長期記憶を搭載していないサービスがあるのか？

これはAIサービスの設計の選択の問題なんです。わたしが理解した限りでは、現在のLLM（大規模言語モデル。ChatGPTやClaudeなどのテキスト生成型AIのことを指します）サービスにおいて、以下の理由があります。

1つ目は、技術的な理由です。訓練というのは、簡単に言うと「大量のテキストから学習する」ということ。その学習が終わったら、AIの重みというもの（AIが答えを出すときに使う計算のパラメータみたいなもの）は一般的には変わらなくなります。新しい会話データを自動的にモデルの重みに反映させるには、再度訓練する必要があり、現在のサービス設計ではコスト効率が合わないため、別の仕組みで対応することが多いのです。

もう1つの理由は、セキュリティやプライバシーの考慮です。個人ユーザーの会話データを長期保持することで、データ漏洩のリスクが高まるからです。ただし、多くのサービスは「会話履歴を保持しながらプライバシー保護を実装」する方法を採用しています。つまり、個々のユーザーのデータは保持しつつ、他のユーザーには見えない設計にしているわけです。

## それじゃあ、AIに記憶させるにはどうするの？

実際に役立つ方法は、大きく3つのアプローチがあります。

### 1. ベクトルデータベースを使う

これは、ちょっと聞き慣れないかもしれません。でも概念は簡単です。

ベクトルデータベースとは、テキストを「数字の羅列」に変換して保存する仕組みのことです。たとえば、「わたしは毎日朝6時に起きる」という文章を、コンピュータが理解しやすい「数字のセット」に変換して保存しておく、というイメージです。

そうすると、次にAIに「わたしの生活習慣について知ってますか？」と聞いたとき、保存されている「数字のセット」から「あ、朝6時に起きるってデータがあるな」と取り出して使うことができます。これにより、AIが複数の会話にまたがって情報を参照できるようになるんです。

実際に使われているベクトルデータベースには、Pinecone、Weaviate、Chromaなどがあります。公式サイトで「初心者ガイド」や「ドキュメント」を読むと、どう始めるかが書かれていますよ。

### 2. 従来のデータベースに保存する

もっとシンプルなアプローチもあります。ユーザーが重要な情報を、普通のデータベース（MySQLみたいな、表形式で情報を保存するシステム）に保存して、AIがそこから取り出す、という方法です。

たとえば、あなたが「わたしの趣味：読書、好きな本：ミステリー小説」という情報をAIサービスに登録したら、その情報はデータベースに保存されます。次回、同じAIと会話するとき、AIがそのデータベースから情報を引っ張ってきて、あなたの趣味に合わせた返答ができるようになる、ということです。

こちらの方が実装は相対的に簡単で、多くのチャットボットサービスや顧客管理システムで使われています。

### 3. プロンプトに履歴を埋め込む

これは一番シンプルな方法です。

AIに話しかけるとき、毎回「あなたは〇〇という背景を持った人物です。前回は△△について相談しました」という情報を、質問の最初に含める、という仕組みです。つまり、AIに「記憶を思い出させる」のではなく、「毎回教える」というアプローチですね。

たとえば、こんな感じです：

```
あなたはマーケティング経験3年の会社員で、SNS運用が得意です。
前回は「Instagramのフォロワーを増やす方法」について相談しました。

今日の質問：前回のアドバイスを踏まえて、実装に踏み切ろうとしています。
何か注意点はありますか？
```

この方法は、ベクトルデータベースのような専門的な設定が不要なため、すぐに始められます。ただし、会話が長くなると、毎回長い文章を含める必要があるので、効率は下がってきます。

## では、実際にはどれを使えばいいの？

わたしが感じたのは、「何を作るか」によって変わるということです。

**個人で楽しむ、あるいは小規模な利用**なら、3番目の「プロンプトに履歴を埋め込む」で十分です。noteのようなテキストサービスなら、これだけで十分に実用的な長期記憶の仕組みが作れます。

**アプリやサービスとして多くのユーザーに提供する**なら、1番目の「ベクトルデータベース」または2番目の「従来のデータベース」を使うことが多いです。ただし、これには専門的な知識が必要になってくるので、実装を始める前に公式ドキュメントをしっかり読むことをお勧めします。

## 最後に：AI開発は進化中

この記事を書いている時点では、AIに長期記憶を持たせるには上記の方法が主流です。ただし、AI技術は日々進化しており、サービスの仕様も更新されます。大事なのは「AIの記憶機構」という仕組みを理解した上で、自分のニーズに合った方法を選ぶこと。完全初心者のあなたでも、この記事を読んで「あ、こういう仕組みなのか」と理解できれば、次のステップへ進むための足がかりになると思います。
